# Exercise 1 — Implementing the Singleton Pattern

## Objective

Implement the Singleton design pattern to ensure that only one instance of a class is created throughout the application.

## Concepts Used

- Singleton pattern
- Private constructor
- Static instance
- Global access point
- Lazy initialization

## How to Run

```text
java SingletonDemo.java
```

## Learning Outcome

After completing this exercise, I understood how to restrict object creation and provide a single shared instance across the application.
